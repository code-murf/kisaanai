## 🔗 Step-by-Step Instructions

### Step 1: Access AWS Builder Center

**Option A: Direct Link**
1. Go to: https://community.aws
2. Or search Google for "AWS Builder Center"

**Option B: From AWS Console**
1. Go to: https://aws.amazon.com
2. Click "Resources" → "Community"
3. Click "AWS Builder Center"

### Step 2: Sign In

1. Click **"Sign In"** (top right)
2. Use your AWS account credentials
3. If you don't have an AWS account:
   - Click "Create AWS Account"
   - Follow registration process
   - Return to community.aws

### Step 3: Create New Post

1. After signing in, click **"Create"** or **"Write a Post"**
2. Select post type: **"Blog Post"** or **"Article"**
3. You'll see the editor interface

### Step 4: Add Title

**Copy this title**:
```
Building KisaanAI: Empowering 100M+ Indian Farmers with AWS AI Services
```

Or use alternative titles:
- "How We Built a Voice-First Agricultural Platform with AWS Bedrock"
- "KisaanAI: Production-Ready AI for Indian Farmers Using 5 AWS Services"
- "From Idea to Production: Building KisaanAI with AWS AI Services"

### Step 5: Add Content

1. Open `BLOG_POST_AWS_BUILDER_CENTER.md`
2. **Copy ALL content** (Ctrl+A, Ctrl+C)
3. **Paste into editor** (Ctrl+V)
4. The markdown formatting should be preserved

**If formatting is lost**:
- Use the editor's formatting tools
- Add headings manually (H1, H2, H3)
- Format code blocks with the code button
- Add bullet points and numbered lists

### Step 6: Add Tags

Add these tags to help people find your post:

**Required Tags**:
- `AWS Bedrock`
- `Amazon S3`
- `Amazon CloudWatch`
- `AWS Transcribe`
- `Amazon EC2`

**Additional Tags**:
- `AI`
- `Machine Learning`
- `Agriculture`
- `India`
- `Hackathon`
- `FastAPI`
- `Next.js`
- `Production Deployment`

### Step 7: Add Images/Screenshots (Optional but Recommended)

**Screenshots to Add**:

1. **Homepage Screenshot**
   - Visit: https://kisaanai.duckdns.org
   - Take screenshot (Win+Shift+S)
   - Add as featured image or in content

2. **Architecture Diagram**
   - Create simple diagram showing AWS services
   - Or use the ASCII diagram from blog

3. **Performance Metrics**
   - Screenshot of test results
   - Or create table in editor

4. **Code Examples**
   - Already included in blog post
   - Ensure they're in code blocks

**How to Add Images**:
1. Click "Insert Image" button in editor
2. Upload screenshot from your computer
3. Add alt text (e.g., "KisaanAI Homepage")
4. Position image in content

### Step 8: Add Metadata

**Author Information**:
- Name: Your name or "KisaanAI Team"
- Bio: "Building AI solutions for Indian agriculture"

**Post Settings**:
- Category: "AI & Machine Learning" or "Solutions"
- Visibility: Public
- Comments: Enabled
- Publication Date: Immediate

### Step 9: Preview

1. Click **"Preview"** button
2. Review the entire post
3. Check:
   - ✅ Title is clear
   - ✅ Formatting is correct
   - ✅ Code blocks are readable
   - ✅ Links work (test them)
   - ✅ Images display properly
   - ✅ No typos

### Step 10: Publish

1. Click **"Publish"** button
2. Confirm publication
3. Wait for confirmation message

### Step 11: Get Link

1. After publishing, you'll see the live post
2. **Copy the URL** from browser address bar
3. It will look like:
   ```
   https://community.aws/posts/building-kisaanai-empowering-farmers
   ```
4. **Save this link** - you need it for hackathon submission!

## ❓ Troubleshooting

### Problem: Can't find "Create Post" button
**Solution**: 
- Make sure you're signed in
- Look for "Write", "Create", or "+" button
- Try refreshing the page

### Problem: Formatting is lost when pasting
**Solution**:
- Use the editor's formatting tools
- Manually add headings (H1, H2, H3)
- Use code block button for code
- Use bullet point button for lists

### Problem: Images won't upload
**Solution**:
- Reduce image size (< 5MB)
- Use JPG or PNG format
- Try uploading one at a time
- Skip images if time is short (not required)

### Problem: Can't publish (button disabled)
**Solution**:
- Add a title
- Add some content (minimum length)
- Add at least one tag
- Check for error messages

### Problem: Link doesn't work after publishing
**Solution**:
- Wait 1-2 minutes for processing
- Refresh the page
- Check if post is set to "Public" not "Draft"
- Try accessing in incognito window
